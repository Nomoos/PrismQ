# Model Issues

This directory tracks issues for the Model module.

## Structure

- **new/** - New issues that haven't been started
- **wip/** - Work in progress issues
- **done/** - Completed issues

## Creating Issues

Create issue files using the format: `YYYY-MM-DD-short-description.md`

When starting work on an issue, move it from `new/` to `wip/`.
When complete, move it from `wip/` to `done/`.
