## Repository Details

### PrismQ.Idea.Sources
**Central Documentation Hub & Python Data Models Library**

- **Repository**: https://github.com/Nomoos/PrismQ.Idea.Sources
- **Type**: Python Library & Documentation
- **Access**: Private
- **Purpose**: Serves as the entry point for the PrismQ ecosystem and provides data models for content sources
- **Key Features**:
  - Python data models (Baseline, InspirationMetrics)
  - Comprehensive documentation hub
  - Ecosystem navigation and taxonomy
- **Key Documents**:
  - [README.md](../README.md) - Overview
  - [TAXONOMY.md](./TAXONOMY.md) - Ecosystem taxonomy
  - [NAVIGATION.md](./NAVIGATION.md) - This document

### PrismQ.IdeaCollector
**Standalone Python CLI for Idea Collection**

- **Repository**: https://github.com/Nomoos/PrismQ.IdeaCollector
- **Type**: Python Application
- **Access**: Public
- **Purpose**: Gathers idea inspirations from various sources
- **Key Features**:
  - CLI interface for idea collection
  - Configurable data sources
  - Metrics tracking
  - Cross-platform support
- **Getting Started**:
  - [README.md](https://github.com/Nomoos/PrismQ.IdeaCollector#readme)
  - [Documentation](https://github.com/Nomoos/PrismQ.IdeaCollector/tree/main/docs)
  - [Contributing Guide](https://github.com/Nomoos/PrismQ.IdeaCollector/blob/main/docs/CONTRIBUTING.md)

### PrismQ.RepositoryTemplate
**Standardized Template for New Projects**

- **Repository**: https://github.com/Nomoos/PrismQ.RepositoryTemplate
- **Type**: Template Repository
- **Access**: Public
- **Purpose**: Provides standardized structure for new PrismQ projects
- **Key Features**:
  - Pre-configured project structure
  - Standard configuration files
  - Proprietary licensing template
  - Python project setup
- **Usage**: Use as template when creating new repositories

## Getting Started with PrismQ

### For New Users
1. Start here: [PrismQ.Idea.Sources README](../README.md)
2. Review the [Taxonomy](./TAXONOMY.md)
3. Explore [PrismQ.IdeaCollector](https://github.com/Nomoos/PrismQ.IdeaCollector)

### For Contributors
1. Review contribution guidelines in each repository
2. Check [PrismQ.RepositoryTemplate](https://github.com/Nomoos/PrismQ.RepositoryTemplate) for standards
3. Follow established patterns and documentation requirements

### For Developers
1. Clone the repository you want to work with
2. Review its README and documentation
3. Set up the development environment as described
4. Check for open issues and contribution opportunities
